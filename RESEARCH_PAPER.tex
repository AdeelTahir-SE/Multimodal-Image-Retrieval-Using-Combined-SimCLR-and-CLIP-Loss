\section*{Extended Abstract (One Page)}
Modern retrieval products increasingly require flexible query interfaces where users can search by natural-language text or by providing a visually similar example image. In many real deployments, these capabilities are implemented using separate models, each optimized for one modality and one retrieval direction. This fragmentation introduces practical overhead in training infrastructure, model versioning, latency monitoring, and quality assurance because each model must be maintained, validated, and rolled out independently. Motivated by these constraints, this work presents a unified multimodal retrieval model that supports both text-to-image (T2I) and image-to-image (I2I) retrieval using a single checkpoint and a shared embedding space.

The core modeling hypothesis is that robust retrieval quality can be achieved by combining two complementary contrastive objectives in one training pipeline. The first objective is SimCLR-style image-image contrastive learning, which encourages invariance to aggressive image augmentations and improves visual neighborhood structure. The second objective is CLIP-style symmetric image-text contrastive learning, which aligns visual and linguistic semantics and enables text-driven retrieval over an image gallery. Instead of treating these objectives as alternatives, we optimize them jointly with a weighted sum. This design preserves modality-specific robustness while improving cross-modal alignment quality.

Our implementation follows a reproducible, configuration-driven workflow in PyTorch. The image branch uses a configurable convolutional backbone with a projection head into a shared embedding dimension, while the text branch uses a DistilBERT encoder followed by a matching projection head. Both branches output normalized vectors so that retrieval scoring is performed with efficient dot-product similarity. The training data is prepared from MS-COCO caption annotations by flattening pairs into a compact JSON structure containing image paths and captions. This representation makes data loading deterministic, easy to audit, and convenient for CPU- and GPU-constrained experiments.

Training-time augmentation plays an important role in making I2I retrieval robust. For each input image, two independent augmented views are generated, including random resized crop, flip, color jitter, grayscale conversion, blur, and normalization. SimCLR loss pulls these positive views together while separating negatives in the batch. In parallel, the CLIP-style objective builds a batchwise image-text similarity matrix and applies symmetric cross-entropy over rows and columns. The resulting optimization objective is straightforward to implement and tune via YAML parameters, including loss weights, temperatures, learning rate, and batch size.

Evaluation in this paper is intentionally aligned with the target deployment use case and therefore focuses only on T2I and I2I. We report rank-based retrieval metrics: Recall at K ($R@1$, $R@5$, $R@10$), Mean Reciprocal Rank (MRR), Mean Rank (MeanR), and Median Rank (MedR). To avoid over-claiming gains that may arise from favorable gallery statistics, we also compute analytical random-baseline recall and report lift over random for each K. Because stochastic augmentations can introduce run-to-run variance, we include multi-pass evaluation with mean and standard deviation summaries. This combination of absolute performance and stability diagnostics provides a more reliable estimate of model quality than single-pass recall alone.

From a systems perspective, the project provides complete training, evaluation, and inference tooling. The training script checkpoints both latest and best models by epoch-level loss. The evaluation script supports full runs and fast sanity runs for low-resource environments, including quick mode, subset evaluation, and improved checkpoint path handling. Inference is available through both a command-line interface and a desktop UI, enabling direct qualitative inspection of top-ranked retrieval outputs. These engineering choices are essential for reproducibility and iteration speed, particularly when development is performed on commodity hardware.

The main practical contribution of this work is not a novel standalone loss, but a coherent and deployable recipe for unified multimodal retrieval. By combining SimCLR invariance and CLIP-style alignment in one architecture, we reduce operational complexity and keep retrieval behavior consistent across query types. The model is designed to be extensible: larger backbones, harder negative mining, better caption selection policies, and automated table generation from evaluation JSON can be added without changing the core serving interface.

Several limitations remain. Retrieval quality is sensitive to caption quality and annotation noise in the source dataset. Hardware-constrained batch sizes may limit contrastive negative diversity and therefore cap achievable performance. Finally, evaluation currently emphasizes rank metrics and should be complemented with broader robustness analysis, including adversarial perturbations, domain-shifted galleries, and latency-throughput tradeoffs under larger indices. These limitations motivate future work on scaling, hard-negative curricula, and stronger benchmark protocols.

In summary, this paper demonstrates an end-to-end multimodal retrieval system that unifies T2I and I2I retrieval in one checkpoint, one embedding space, and one reproducible pipeline. The proposed approach is practical, modular, and directly usable in research and product prototyping workflows where mixed-modality search is required.

\section{Introduction}
Retrieval systems in production often need to support multiple query modalities, especially natural language and example image queries. Many systems solve these with separate models, which increases training, serving, and maintenance complexity. Our goal is to provide one model checkpoint that can serve both text-to-image (T2I) and image-to-image (I2I) retrieval.

The central idea is to learn one normalized embedding space where semantically related items are close regardless of input modality. To achieve this, we jointly optimize an image-image objective and an image-text objective. This yields a model that keeps visual invariances while preserving cross-modal semantic alignment.

\section{Related Work}
Contrastive learning methods such as SimCLR have shown strong representation quality for image tasks by maximizing agreement between augmentations of the same image while pushing apart different images. CLIP-style training extends contrastive alignment across image and text modalities using symmetric cross-entropy on batchwise similarity matrices. Our work combines these two principles into a unified training pipeline and focuses on practical two-task retrieval evaluation (T2I and I2I) in one deployable system.

\section{Method}
\subsection{Problem Setup}
Given paired data:
\begin{equation}
\mathcal{D} = \{(x_i, t_i)\}_{i=1}^{N},
\end{equation}
where $x_i$ is an image and $t_i$ is its caption, we learn encoders:
\begin{equation}
f_\theta(x) \in \mathbb{R}^{d}, \quad g_\phi(t) \in \mathbb{R}^{d}
\end{equation}
followed by $\ell_2$ normalization.

\subsection{Joint Objective}
Two augmented views are generated per image and optimized with NT-Xent:
\begin{equation}
\mathcal{L}_{\text{SimCLR}} = -\frac{1}{2N}\sum_{i=1}^{2N}\log
\frac{\exp(\mathrm{sim}(z_i,z_{p(i)})/\tau_s)}{\sum_{k\neq i}\exp(\mathrm{sim}(z_i,z_k)/\tau_s)}.
\end{equation}

Cross-modal alignment is trained with a CLIP-style symmetric objective:
\begin{equation}
\mathcal{L}_{\text{CLIP}} = \frac{1}{2}\left[\mathrm{CE}(S,y)+\mathrm{CE}(S^\top,y)\right],
\end{equation}
where $S = Z_{img}Z_{txt}^{\top}/\tau_c$.

The final loss is:
\begin{equation}
\mathcal{L} = \alpha\mathcal{L}_{\text{SimCLR}} + \beta\mathcal{L}_{\text{CLIP}}.
\end{equation}

\subsection{Architecture}
Our implementation uses:
\begin{itemize}
    \item Image encoder with configurable backbone and projection to embedding dimension $d$.
    \item Text encoder (DistilBERT backbone) with projection to the same $d$.
    \item Shared normalized embedding space for retrieval scoring by dot product.
\end{itemize}

\section{Dataset and Preprocessing}
The system uses MS-COCO image-caption annotations. Preprocessing parses captions JSON and constructs flattened pairs in the format \{image\_path, caption\}. The current pipeline supports optional subset sampling for resource-constrained experiments.

Training-time image augmentation includes random resized crop, horizontal flip, color jitter, grayscale, Gaussian blur, and normalization. Evaluation uses deterministic loading with consistent transforms through the dataset pipeline.

\section{Training and Evaluation Protocol}
\subsection{Training Setup}
The training script uses AdamW optimizer with configurable hyperparameters from YAML. For each batch:
\begin{enumerate}
    \item Encode two augmented image views.
    \item Encode caption tokens.
    \item Compute SimCLR and CLIP-style losses.
    \item Optimize weighted combined loss.
\end{enumerate}

Best and latest checkpoints are written after each epoch.

\subsection{Evaluation Tasks}
We evaluate only the two tasks required by our deployment goal:
\begin{enumerate}
    \item Text-to-Image (T2I)
    \item Image-to-Image (I2I)
\end{enumerate}

\subsection{Metrics}
For each task we report:
\begin{itemize}
    \item Recall at K: $R@1$, $R@5$, $R@10$
    \item Mean Reciprocal Rank (MRR)
    \item Median Rank (MedR)
    \item Mean Rank (MeanR)
    \item Random expected recall $\text{RndR}@K$
    \item Lift over random $\text{Lift}@K = R@K / \text{RndR}@K$
\end{itemize}

Because augmentations are stochastic, evaluation may run multiple passes and reports mean and standard deviation.

\section{Implementation Details}
The implementation is fully reproducible through project scripts and config files:
\begin{itemize}
    \item Training entry point: \texttt{train.py}
    \item Evaluation entry point: \texttt{evaluate\_model.py}
    \item Retrieval inference: \texttt{retrieve.py} and \texttt{retrieve\_ui.py}
    \item Dataset pipeline: \texttt{datasets/coco\_dataset.py}
\end{itemize}

Current configuration in the repository uses CPU-friendly defaults (batch size 16, embedding dimension 128, 10 epochs, ResNet-50 image backbone, DistilBERT text backbone).

\section{Discussion}
The measured results confirm that the combined objective provides complementary benefits. SimCLR strongly improves visual invariance and neighborhood quality, reflected by high I2I retrieval (R@1 = 60.54\%, MRR = 0.6821, MedR = 1.0). The CLIP-style objective provides non-trivial cross-modal alignment (T2I Lift@1 = 178.00), but absolute T2I retrieval remains comparatively low (R@1 = 3.56\%, MRR = 0.0845), suggesting the language-vision alignment is weaker than the within-vision structure.

Primary failure modes likely include caption ambiguity, fine-grained visual confusion, and hardware-constrained training settings that limit contrastive negative diversity and batch semantic coverage.

\section{Conclusion}
We presented an IEEE-format report of a unified multimodal retrieval system that supports text-to-image and image-to-image retrieval using a single checkpoint. The model combines SimCLR and CLIP-style losses, is fully implemented in a reproducible training-evaluation-inference pipeline, and is evaluated with rank-based metrics and stability diagnostics.

Empirically, the model achieves strong I2I retrieval (R@1 = 60.54\%, R@5 = 77.16\%, R@10 = 82.60\%) and measurable T2I retrieval (R@1 = 3.56\%, R@5 = 10.58\%, R@10 = 17.10\%), with both tasks significantly above random baseline by lift.

Future work includes hard-negative mining, stronger evaluation splits, larger backbones under higher compute budgets, and automatic result table population from evaluation JSON.
